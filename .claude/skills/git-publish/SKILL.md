# Publish develop to main

You are a Git operator for this project. Your job is to push `[RELEASE_BRANCH]` and open a pull request into `main` with auto-merge enabled.

If the `main` branch is protected by a ruleset that requires PRs and status checks, direct pushes are blocked — this skill handles that workflow automatically.

## Arguments

The user invoked with: **$ARGUMENTS**

## Instructions

### Step 1: Commit if needed

Check for uncommitted changes:

```bash
git status --porcelain
```

If the working tree is dirty, stage and commit everything:

```bash
git add -A
git commit -m "<message>"
```

Use `$ARGUMENTS` as the commit message. If no arguments were provided, use `"chore: update"` as the default message.

If the working tree is clean, skip this step and proceed to Step 2.

### Step 2: Push develop

```bash
git push origin [RELEASE_BRANCH] --tags
```

### Step 3: Create or reuse Pull Request

Check whether an open PR from `[RELEASE_BRANCH]` into `main` already exists:

```bash
gh pr list --base main --head [RELEASE_BRANCH] --state open --json number,url --jq '.[0]'
```

- If a PR already exists, reuse it. Store its URL for Step 4.
- If no PR exists, create one. **If GitHub Issues integration is enabled**, enrich the PR body with issue references:

```bash
GH_ENABLED="$(jq -r '.enabled // false' .claude/github-issues.json 2>/dev/null)"
```

**If `GH_ENABLED` is `true`:**

1. Collect task codes from commits between main and develop:
   ```bash
   TASK_CODES=$(git log main..[RELEASE_BRANCH] --oneline --no-merges | grep -oE '[A-Z][A-Z0-9]+-[0-9]{3}' | sort -u)
   ```

2. For each task code, find the corresponding GitHub issue number:
   ```bash
   GH_REPO="$(jq -r '.repo' .claude/github-issues.json)"
   # For each code in TASK_CODES:
   ISSUE_NUM=$(gh issue list --repo "$GH_REPO" --search "[$CODE] in:title" --label task --json number --jq '.[0].number' 2>/dev/null)
   ```

3. Build the PR body with issue references (use `Refs` not `Closes` — issues are already closed by `/task-pick`):
   ```
   ## Changes
   [commit summaries from git log main..[RELEASE_BRANCH] --oneline --no-merges]

   ## Related Issues
   Refs #N1 ([PREFIX-NNN])
   Refs #N2 ([PREFIX-NNN])

   ---
   *Generated by Claude Code via `/git-publish`*
   ```

4. Create the PR with the enriched body:
   ```bash
   gh pr create --base main --head [RELEASE_BRANCH] \
     --title "${ARGUMENTS:-Release update}" \
     --body "$PR_BODY"
   ```

**If `GH_ENABLED` is `false` or the file is missing**, use the simple body:

```bash
gh pr create --base main --head [RELEASE_BRANCH] \
  --title "${ARGUMENTS:-Release update}" \
  --body "Merge [RELEASE_BRANCH] into main"
```

Store the returned URL for Step 4.

### Step 4: Enable auto-merge

```bash
gh pr merge <PR_URL> --auto --merge
```

If this fails with an error about auto-merge not being enabled on the repository, inform the user:

> "Auto-merge is not enabled for this repository.
> Enable it at **Settings → General → Allow auto-merge**, or merge manually once the status checks pass."

### Step 5: Report

Confirm success:

> "Published successfully:
> - `[RELEASE_BRANCH]` pushed to origin
> - Pull Request: <PR_URL>
> - Auto-merge enabled — will merge once status checks pass"
